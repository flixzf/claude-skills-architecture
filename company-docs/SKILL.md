# 회사 문서 작성 스킬

회사에서 사용하는 공식 문서를 표준 양식에 맞춰 작성하는 스킬입니다.

## 지원 문서 유형

1. **공문/기안서** - 대내외 공식 문서
2. **보고서** - 주간/월간/프로젝트 보고서
3. **회의록** - 회의 내용 기록 및 정리
4. **제안서** - 사업 제안 및 기술 제안
5. **사업계획서** - 프로젝트 계획 수립
6. **표준서/기준서** - 업무 표준 및 기준 정의 (계층적 번호 체계)
7. **설문지** - 설문 조사 및 의견 수렴
8. **품질관리 문서** - QA/QC 관련 문서
9. **안전관리 문서** - 안전 점검 및 보고서

## 작성 지침

### 1. 문서 유형 파악
사용자가 요청한 문서 유형을 명확히 파악하고, 해당하는 템플릿을 참조합니다.

### 2. 기본 정보 수집
다음 정보를 사용자에게 확인합니다:
- 문서 제목
- 작성자/부서
- 수신자 (해당시)
- 작성 일자
- 주요 내용 요약

### 3. 문서 구조
각 문서 유형별 표준 구조를 따릅니다:

**공문/기안서**
```
1. 문서 번호 및 날짜
2. 수신/발신
3. 제목
4. 내용
5. 첨부 서류 (해당시)
```

**보고서**
```
1. 보고서 제목
2. 작성일 및 보고 기간
3. 개요
4. 주요 내용
   - 진행 사항
   - 이슈 사항
   - 향후 계획
5. 결론 및 건의사항
```

**회의록**
```
1. 회의명
2. 일시 및 장소
3. 참석자
4. 회의 안건
5. 논의 내용
6. 결정 사항
7. 조치 사항 (담당자/기한)
```

**제안서**
```
1. 표지 (제목, 제안사, 날짜)
2. 제안 개요
3. 배경 및 필요성
4. 제안 내용
   - 사업 범위
   - 추진 방법
   - 일정 계획
5. 기대 효과
6. 소요 예산
```

**표준서/기준서**
```
계층적 번호 체계 사용:
1. [대분류]
  1.1 [중분류]
    1.1.1 [소분류]
      1.1.1.1 [세부항목]
        1.1.1.1.1 [상세항목]
  1.2 [중분류]
2. [대분류]
  2.1 [중분류]
    2.1.1 [소분류]
```

**설문지**
```
1. 설문 기본 정보
2. 응답자 정보 (선택사항)
3. 설문 안내
4. 설문 문항
   - 객관식 (단일/복수 선택)
   - 척도형 (리커트 척도)
   - 순위형
   - 개방형
   - 매트릭스 (표 형식)
5. 추가 의견
6. 마무리 안내
```

### 4. 작성 원칙

- **명확성**: 간결하고 명확한 표현 사용
- **정확성**: 날짜, 수치, 고유명사 정확히 기재
- **공식성**: 공식 문서 어투 유지
- **논리성**: 체계적이고 논리적인 구성
- **완성도**: 맞춤법, 띄어쓰기 확인

### 5. 건축/건설 분야 특화

건축/건설 분야 문서 작성시 다음 사항을 반영:
- BIM 관련 용어 및 프로세스
- 건축법, 건설산업기본법 등 관련 법규
- 품질/안전 기준 및 규정
- 도면, 시방서 등 기술 문서 참조

## 템플릿 활용

`templates/` 디렉토리의 표준 양식을 참조하여 작성합니다.
사용자 요청에 맞게 템플릿을 커스터마이징하되, 핵심 구조는 유지합니다.

## 예시

**사용자**: "내일 있을 BIM 워크샵 회의록 템플릿 만들어줘"

**응답**:
1. 회의록 템플릿을 확인
2. BIM 워크샵에 맞는 항목 구성
3. 날짜를 내일 날짜로 설정
4. 주요 논의 예상 안건 (BIM 표준, 협업 프로세스 등) 포함
5. 문서 생성

## 주의사항

- 민감한 정보 (개인정보, 영업비밀) 취급시 주의
- 회사 고유의 문서 번호 체계가 있다면 사용자에게 확인
- 법적 검토가 필요한 문서는 담당 부서 검토 권고
- 첨부 파일이나 참조 문서가 있다면 명시

## 출력 형식

**기본 형식: Word (.docx)**

모든 문서는 기본적으로 Word 형식으로 작성합니다.
- docx 스킬을 활용하여 전문적인 문서 생성
- 표, 스타일, 서식 등 Word 기능 활용
- 특별한 요청이 없는 한 .docx 파일로 생성

**기타 형식**
- 요청시: .xlsx (Excel) 형식 - 표 중심 문서 (xlsx 스킬 활용)
- 요청시: .pdf (PDF) 형식 - 최종 배포용 (pdf 스킬 활용)

## 문서 작성 절차

1. **문서 유형 확인** - 사용자 요청 분석
2. **템플릿 선택** - templates/ 디렉토리에서 적절한 템플릿 참조
3. **docx 스킬 활용** - Word 문서로 작성
4. **내용 구성** - 템플릿 기반으로 내용 작성
5. **서식 적용** - 제목, 본문, 표 등 스타일 적용
6. **검토 및 생성** - 최종 검토 후 .docx 파일 생성

## MD → DOCX 변환 도구

템플릿을 마크다운(MD)으로 관리하고 필요시 Word 문서로 변환할 수 있습니다.

### 변환 스크립트 사용법

```bash
# 기본 사용법
python convert_md_to_docx.py templates/설문지.md

# 출력 파일명 지정
python convert_md_to_docx.py templates/설문지.md output/my_survey.docx
```

### 지원 기능

- **제목**: # ~ ###### (6단계)
- **리스트**: 번호 리스트, 불릿 리스트
- **표**: 마크다운 표 형식 자동 변환
- **체크박스**: □, ○ 기호
- **서식**: 굵게(**텍스트**)
- **수평선**: ---, ***

### 템플릿 관리 워크플로우

1. **MD로 템플릿 작성/편집** - IDE에서 직접 편집 가능
2. **버전 관리** - Git으로 변경 이력 추적
3. **필요시 DOCX 변환** - 변환 스크립트 실행
4. **배포** - Word 문서로 배포
