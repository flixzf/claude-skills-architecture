# 프로젝트 보고 프레젠테이션 템플릿

이 템플릿은 pptx 스킬을 사용하여 PowerPoint 형식의 프로젝트 진행 보고 자료를 작성하기 위한 참고 자료입니다.

## 슬라이드 구성 (총 12-18장)

### 슬라이드 1: 표지
**레이아웃**: 제목 전용
**내용**:
- 프로젝트명 진행 보고
- 보고 기간 (예: 2024년 11월)
- 보고자 / 소속
- 보고일
- 회사 로고

**디자인**:
- 프로젝트 관련 이미지 배경
- 제목: 36pt, 굵게
- 보고 기간 강조

---

### 슬라이드 2: 목차
**레이아웃**: 제목 + 본문
**내용**:
```
목차

1. 프로젝트 개요
2. 진행 현황
3. 주요 실적
4. 이슈 및 리스크
5. 향후 계획
```

**디자인**:
- 간결한 목차
- 진행 상태 표시 (완료/진행중)

---

### 슬라이드 3: 프로젝트 개요
**레이아웃**: 제목 + 본문
**내용**:
```
프로젝트 개요

• 프로젝트명: [프로젝트명]
• 발주처: [발주처명]
• 계약 기간: YYYY-MM-DD ~ YYYY-MM-DD
• 프로젝트 규모: [규모]
• 예산: XXX억원
• 프로젝트 목표: [핵심 목표]
```

**디자인**:
- 표 형식 또는 아이콘 + 텍스트
- 핵심 정보 명확히

---

### 슬라이드 4: 프로젝트 구조 (선택)
**레이아웃**: 제목 + 다이어그램
**내용**:
```
프로젝트 조직도

발주처
   │
PM ─── 설계팀
   │
   ├─── 시공팀
   │
   └─── 감리팀
```

**디자인**:
- 조직도 형식
- 역할별 색상 구분

---

### 슬라이드 5: 전체 진행률
**레이아웃**: 제목 + 차트
**내용**:
```
전체 진행 현황

[진행률 게이지 또는 막대 그래프]

• 계획 진행률: XX%
• 실제 진행률: XX%
• 차이: ±X%p

상태: [정상/주의/지연]
```

**디자인**:
- 게이지 차트 또는 진행률 바
- 계획 vs 실제 비교
- 상태별 색상 (정상=초록, 주의=주황, 지연=빨강)

---

### 슬라이드 6: 일정 현황
**레이아웃**: 제목 + 간트 차트
**내용**:
```
프로젝트 일정 현황

[간트 차트]

주요 마일스톤:
✓ 착수 보고: YYYY-MM-DD (완료)
✓ 설계 완료: YYYY-MM-DD (완료)
→ 시공 착수: YYYY-MM-DD (진행중)
• 준공: YYYY-MM-DD (예정)
```

**디자인**:
- 간트 차트
- 완료(초록), 진행중(파랑), 예정(회색)
- 현재 시점 표시

### 슬라이드 7: 단계별 진행 현황

```

|------|---------|---------|------|------|
| 설계 | 100% | 100% | ✓ | 완료 |
| 시공 | 50% | 48% | → | 진행중 |
| 검수 | 0% | 0% | - | 대기 |
```

**디자인**:
- 표 형식
- 상태 아이콘 (✓, →, !)
- 색상 코드

## 스타일 가이드

### 색상
- **정상**: 초록 (#00CC66) - 목표 달성, 정상 진행
- **주의**: 주황 (#FF9900) - 관심 필요
- **경고**: 빨강 (#CC0000) - 즉각 조치 필요
- **중립**: 파랑 (#0066CC) - 일반 정보

### 상태 아이콘
- ✓ : 완료
- → : 진행중
- ! : 주의 필요
- ✗ : 지연/문제
- - : 대기/예정

### 차트
- 진행률: 게이지 차트 또는 진행률 바
- 추이: 꺾은선 그래프
- 비교: 막대 그래프
- 일정: 간트 차트

## 보고 작성 체크리스트

### 내용 확인
- [ ] 진행률이 정확한가?
- [ ] 주요 실적이 포함되었는가?
- [ ] 이슈가 명확히 기술되었는가?
- [ ] 대응 방안이 구체적인가?
- [ ] 향후 계획이 현실적인가?

### 데이터 확인
- [ ] 날짜가 정확한가?
- [ ] 수치가 검증되었는가?
- [ ] 차트가 명확한가?
- [ ] 사진에 날짜가 표기되었는가?

### 발표 준비
- [ ] 보고 시간에 맞는 분량인가?
- [ ] 경영진 질문에 답변 준비했는가?
- [ ] 백업 데이터가 있는가?
- [ ] 보고서(문서) 버전도 준비했는가?

## 보고 발표 팁

### 발표 전
- 경영진 관심사 파악
- 질문 예상 및 답변 준비
- 수치 데이터 검증
- 타이밍 리허설

### 발표 중
- 핵심부터 먼저 (Bottom Line Up Front)
- 문제는 솔루션과 함께
- 긍정적이되 현실적으로
- 질문에 명확히 답변

### 보고 후
- 지시 사항 기록
- 후속 조치 일정 확인
- 회의록 작성 및 공유
- 다음 보고 일정 확인
