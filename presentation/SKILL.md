# 프레젠테이션 작성 스킬

회사에서 사용하는 PowerPoint 프레젠테이션을 체계적으로 작성하는 스킬입니다.

## 지원 프레젠테이션 유형

1. **기획 보고** - 건축 기획, 사업성 분석, 타당성 검토 (기획팀 특화)
2. **사업 제안 발표** - 신규 사업 제안 및 투자 유치
3. **기술 발표** - BIM, 건축 기술, 신기술 소개
4. **프로젝트 보고** - 진행 상황 및 실적 보고
5. **BIM 워크샵** - BIM 교육 및 협업 세미나
6. **킥오프 미팅** - 프로젝트 착수 및 계획 공유
7. **실적 발표** - 분기별/연간 성과 보고
8. **대외 홍보** - 회사 소개 및 홍보 자료
9. **품질/안전 교육** - 교육 및 훈련 자료

## 작성 지침

### 1. 프레젠테이션 유형 파악
사용자가 요청한 발표 유형을 명확히 파악하고, 해당하는 템플릿을 참조합니다.

### 2. 기본 정보 수집
다음 정보를 사용자에게 확인합니다:
- 발표 제목
- 발표자/부서
- 발표 일시 및 장소
- 청중 (임원진, 고객사, 내부 직원 등)
- 발표 시간 (슬라이드 분량 결정에 중요)
- 주요 전달 메시지

### 3. 슬라이드 구조

**기본 구조**
```
1. 표지 슬라이드
   - 발표 제목
   - 발표자/부서
   - 발표 일시
   - 회사 로고

2. 목차 슬라이드
   - 주요 섹션 나열
   - 발표 흐름 안내

3. 개요 슬라이드
   - 배경
   - 목적
   - 범위

4. 본론 슬라이드들
   - 주요 내용
   - 데이터/차트
   - 이미지/다이어그램

5. 결론 슬라이드
   - 핵심 메시지 요약
   - 기대 효과
   - 향후 계획

6. Q&A 슬라이드
   - 질의응답 안내
   - 연락처 정보
```

**사업 제안 발표**
```
1. 표지
2. 목차
3. 제안 배경 및 필요성
4. 현황 분석
5. 제안 내용
   - 추진 방향
   - 세부 계획
   - 기대 효과
6. 추진 일정
7. 소요 예산
8. 추진 체계
9. 결론
10. Q&A
```

**기술 발표**
```
1. 표지
2. 목차
3. 기술 개요
4. 기술적 배경
5. 핵심 기술 설명
6. 적용 사례
7. 기대 효과
8. 향후 계획
9. Q&A
```

**프로젝트 보고**
```
1. 표지
2. 목차
3. 프로젝트 개요
4. 진행 현황
   - 일정 진행률
   - 주요 실적
5. 이슈 및 리스크
6. 향후 계획
7. 결론
8. Q&A
```

### 4. 슬라이드 작성 원칙

**1슬라이드 1메시지 원칙**
- 각 슬라이드는 하나의 핵심 메시지만 전달
- 제목에 핵심 메시지 반영
- 너무 많은 정보를 한 슬라이드에 담지 않기

**시각적 계층 구조**
- 제목 > 주요 내용 > 세부 내용
- 중요도에 따른 크기와 색상 차별화
- 여백을 활용한 가독성 확보

**텍스트 최소화**
- 키워드 중심으로 작성
- 문장보다는 단어나 구문 사용
- 자세한 설명은 발표자 노트에 작성

**일관성 유지**
- 폰트, 색상, 레이아웃 일관성
- 슬라이드 마스터 활용
- 회사 브랜드 가이드 준수

**시각 자료 활용**
- 데이터는 표보다 차트로
- 복잡한 개념은 다이어그램으로
- 고품질 이미지 사용

### 5. 건축/건설 분야 특화

건축/건설 분야 프레젠테이션 작성시 다음 사항을 반영:
- BIM 모델 3D 이미지 활용
- 도면, 단면도, 상세도 삽입
- LOD 단계별 시각화
- 프로젝트 타임라인 (간트 차트)
- 시공 단계별 사진 자료
- 법규 및 기준 참조 (건축법, KDS 등)
- 품질/안전 지표 시각화

## 레이아웃 유형

### 기본 레이아웃
1. **제목 + 본문** - 일반적인 내용 전달
2. **제목 + 2단 구성** - 비교 내용
3. **제목 + 이미지** - 시각 자료 중심
4. **제목 + 표/차트** - 데이터 중심
5. **전체 이미지** - 임팩트 있는 시각 자료
6. **빈 슬라이드** - 섹션 구분

### 데이터 시각화
- **막대 그래프**: 항목 간 비교
- **꺾은선 그래프**: 추세 및 변화
- **원 그래프**: 비율 및 구성
- **누적 그래프**: 누적 효과
- **산점도**: 상관 관계
- **간트 차트**: 일정 관리

## 템플릿 활용

`templates/` 디렉토리의 표준 양식을 참조하여 작성합니다.
사용자 요청에 맞게 템플릿을 커스터마이징하되, 핵심 구조는 유지합니다.

## 예시

**사용자**: "내일 있을 BIM 워크샵 발표 자료 만들어줘. 1시간 발표 예정이야."

**응답**:
1. BIM 워크샵 템플릿 확인
2. 1시간 발표 → 약 20-30장 분량 계획
3. 워크샵 주요 내용 (BIM 개요, 협업 프로세스, 실습 안내) 구성
4. 실습 가이드 슬라이드 추가
5. 날짜를 내일 날짜로 설정
6. 문서 생성

## 주의사항

- 발표 시간 고려 (1분당 1-2장 기준)
- 청중 수준에 맞는 내용 구성
- 애니메이션은 최소화 (집중 방해 요소)
- 백업 슬라이드 준비 (Q&A 대비 상세 자료)
- 회사 기밀 정보 취급 주의
- 저작권 있는 이미지 사용 시 출처 표기

## 출력 형식

**기본 형식: PowerPoint (.pptx)**

모든 프레젠테이션은 기본적으로 PowerPoint 형식으로 작성합니다.
- pptx 스킬을 활용하여 전문적인 프레젠테이션 생성
- 슬라이드 마스터, 레이아웃, 테마 활용
- 차트, 표, 도형, 이미지 등 다양한 요소 활용
- 특별한 요청이 없는 한 .pptx 파일로 생성

**기타 형식**
- 요청시: .pdf (PDF) 형식 - 배포용 (pdf 스킬 활용)
- 요청시: Google Slides 형식 - 온라인 협업용

## 프레젠테이션 작성 절차

1. **발표 유형 확인** - 사용자 요청 분석
2. **청중 및 목적 파악** - 발표 대상과 목적 확인
3. **템플릿 선택** - templates/ 디렉토리에서 적절한 템플릿 참조
4. **pptx 스킬 활용** - PowerPoint 문서로 작성
5. **슬라이드 구성** - 템플릿 기반으로 내용 작성
6. **레이아웃 및 디자인 적용** - 시각적 요소 배치
7. **검토 및 생성** - 최종 검토 후 .pptx 파일 생성

## 색상 및 디자인 가이드

### 색상 사용
- **주 색상**: 회사 브랜드 컬러
- **강조 색상**: 중요 정보 강조 (빨강, 주황)
- **배경 색상**: 흰색 또는 밝은 회색 (가독성 우선)
- **텍스트 색상**: 진한 회색 또는 검정 (명암 대비)

### 폰트 사용
- **제목**: 24-40pt, 굵게
- **본문**: 18-24pt, 보통
- **주석/캡션**: 14-16pt, 가늘게
- **한글 폰트**: 맑은 고딕, 나눔고딕
- **영문 폰트**: Arial, Calibri

### 이미지 가이드
- 최소 해상도: 150 dpi
- 이미지 크기: 슬라이드의 40-60% 권장
- 출처 표기: 하단 작은 글씨로
- 건축 도면: 범례 포함 필수

## 발표 시간별 슬라이드 분량 가이드

| 발표 시간 | 권장 슬라이드 수 | 비고 |
|-----------|------------------|------|
| 5분 | 5-8장 | 핵심만 간략히 |
| 10분 | 10-15장 | 주요 내용 중심 |
| 20분 | 15-25장 | 표준 발표 |
| 30분 | 20-35장 | 상세 설명 포함 |
| 1시간 | 30-50장 | 질의응답 시간 포함 |

※ 1분당 1-2장 기준, 슬라이드 복잡도에 따라 조정

## 체크리스트

발표 자료 완성 후 확인 사항:
- [ ] 모든 슬라이드에 제목이 있는가?
- [ ] 슬라이드 번호가 표시되는가?
- [ ] 회사 로고가 일관되게 표시되는가?
- [ ] 폰트와 색상이 일관성 있는가?
- [ ] 차트와 표의 범례가 명확한가?
- [ ] 이미지 해상도가 충분한가?
- [ ] 텍스트가 너무 많지 않은가? (6줄 이내 권장)
- [ ] 맞춤법과 띄어쓰기가 정확한가?
- [ ] 발표 시간에 적절한 분량인가?
- [ ] Q&A 슬라이드가 있는가?
- [ ] 연락처 정보가 포함되어 있는가?
