# 프레젠테이션 작성 참고 가이드

## 프레젠테이션 디자인 원칙

### 1. 가독성 (Readability)
- **폰트 크기**: 최소 18pt 이상 (청중 고려)
- **색상 대비**: 배경과 텍스트의 명암 대비 확보
- **줄 간격**: 1.5-2배 권장
- **여백**: 슬라이드 여백 충분히 확보 (답답함 방지)

**좋은 예**:
```
제목: 32pt, 굵게, 진한 회색
본문: 20pt, 보통, 검정
배경: 흰색
```

**나쁜 예**:
```
제목: 12pt (너무 작음)
본문: 10pt (읽기 어려움)
배경: 노란색, 텍스트: 흰색 (대비 부족)
```

### 2. 일관성 (Consistency)
- **슬라이드 마스터 활용**: 모든 슬라이드에 동일한 레이아웃 적용
- **폰트 통일**: 최대 2-3종류의 폰트만 사용
- **색상 팔레트**: 주 색상 3-4가지로 제한
- **아이콘 스타일**: 동일한 스타일의 아이콘 사용

### 3. 간결성 (Simplicity)
- **1슬라이드 1메시지**: 한 슬라이드에 하나의 핵심만
- **텍스트 최소화**: 6줄 이내, 각 줄은 6-8단어 이내 (6x6 규칙)
- **불필요한 요소 제거**: 장식적 요소 최소화
- **키워드 중심**: 문장보다는 핵심 키워드 나열

### 4. 흐름성 (Flow)
- **논리적 순서**: 도입 → 전개 → 결론
- **스토리라인**: 청중이 따라갈 수 있는 이야기 구성
- **전환 슬라이드**: 섹션 전환 시 구분 슬라이드 삽입
- **진행 표시**: 목차 슬라이드에서 현재 위치 표시

## 슬라이드 유형별 레이아웃

### 1. 표지 슬라이드
**구성 요소**:
```
상단: 회사 로고
중앙: 발표 제목 (큰 폰트, 굵게)
하단: 발표자명, 부서, 날짜
```

**디자인 팁**:
- 제목은 슬라이드의 1/3 크기 차지
- 배경 이미지 사용 시 투명도 조절로 텍스트 가독성 확보
- 회사 브랜드 컬러 활용

### 2. 목차 슬라이드
**구성 요소**:
```
제목: "목차" 또는 "Contents"
본문:
  1. 섹션 1
  2. 섹션 2
  3. 섹션 3
  ...
```

**디자인 팁**:
- 번호 또는 아이콘으로 섹션 구분
- 현재 진행 위치 표시 (색상 강조)
- 간결하게 3-5개 섹션 권장

### 3. 제목 + 본문 슬라이드
**구성 요소**:
```
제목: [핵심 메시지]
본문:
  • 포인트 1
  • 포인트 2
  • 포인트 3
```

**디자인 팁**:
- 불릿 포인트 3-6개 권장
- 계층 구조는 2단계까지만
- 중요 단어는 굵게 또는 색상 강조

### 4. 제목 + 2단 구성 슬라이드
**구성 요소**:
```
제목: [비교 내용]
왼쪽:          오른쪽:
• 항목 1      • 항목 1
• 항목 2      • 항목 2
```

**디자인 팁**:
- Before/After 비교
- 장단점 비교
- 두 가지 옵션 비교
- 세로선으로 구분

### 5. 제목 + 이미지 슬라이드
**구성 요소**:
```
제목: [이미지 설명]
이미지: 슬라이드의 70-80% 차지
캡션: 이미지 설명 또는 출처
```

**디자인 팁**:
- 고해상도 이미지 사용
- 이미지가 주인공, 텍스트는 최소화
- 이미지 주변에 여백 확보

### 6. 제목 + 차트/표 슬라이드
**구성 요소**:
```
제목: [데이터 설명 및 결론]
차트/표: 중앙 배치
주석: 핵심 수치 강조
```

**디자인 팁**:
- 차트 범례 명확히
- 핵심 데이터 색상 강조
- 표는 5행 이내 권장
- 출처 표기

## 색상 가이드

### 기본 색상 팔레트
```
주 색상 (Primary): 회사 브랜드 컬러
보조 색상 (Secondary): 주 색상의 보색 또는 유사색
강조 색상 (Accent): 빨강, 주황 (경고, 중요)
배경 색상 (Background): 흰색, 밝은 회색
텍스트 색상 (Text): 검정, 진한 회색
```

### 건축/건설 분야 추천 색상
- **주 색상**: 청록색 (#0066CC), 네이비 (#003366)
- **보조 색상**: 회색 (#666666), 연한 회색 (#CCCCCC)
- **강조 색상**: 주황 (#FF6600), 빨강 (#CC0000)
- **배경**: 흰색 (#FFFFFF), 밝은 회색 (#F5F5F5)

### 색상 사용 원칙
1. **최대 5가지 색상**: 너무 많은 색상은 산만함
2. **대비 확보**: WCAG 2.0 AA 기준 준수 (4.5:1)
3. **색맹 고려**: 빨강-초록 조합 지양
4. **일관성**: 동일한 의미는 동일한 색상 (예: 위험=빨강)

### 데이터 시각화 색상
- **단일 계열**: 같은 색상의 명도 변화
- **다중 계열**: 구분되는 색상 사용
- **긍정/부정**: 파랑/빨강, 초록/빨강
- **중립**: 회색

## 폰트 가이드

### 한글 폰트
**프레젠테이션 권장 폰트**:
1. **맑은 고딕** - 가독성 우수, 기본 폰트
2. **나눔스퀘어** - 모던하고 깔끔
3. **나눔고딕** - 공식 문서에 적합
4. **본고딕** - 세련된 느낌

**사용 지양 폰트**:
- 궁서체 (고루함)
- 필기체 (가독성 낮음)
- 장식적 폰트 (프로페셔널하지 않음)

### 영문 폰트
**프레젠테이션 권장 폰트**:
1. **Arial** - 깔끔하고 읽기 쉬움
2. **Calibri** - 모던하고 부드러움
3. **Helvetica** - 전문적이고 세련됨
4. **Open Sans** - 웹 친화적

### 폰트 크기 가이드
| 요소 | 크기 | 스타일 |
|------|------|--------|
| 슬라이드 제목 | 32-40pt | 굵게 |
| 섹션 제목 | 28-32pt | 굵게 |
| 본문 텍스트 | 20-24pt | 보통 |
| 불릿 포인트 | 18-22pt | 보통 |
| 캡션/주석 | 14-16pt | 가늘게 |
| 차트 레이블 | 16-18pt | 보통 |

### 폰트 사용 원칙
- **최대 2종류**: 제목용 + 본문용
- **계층 구조**: 크기와 굵기로 차별화
- **가독성 우선**: 장식보다는 명확성
- **일관성**: 동일한 요소는 동일한 폰트

## 데이터 시각화 가이드

### 1. 차트 선택 기준

#### 막대 그래프 (Bar Chart)
**사용 시기**:
- 항목 간 크기 비교
- 카테고리별 수치 비교
- 순위 표현

**예시**: 부서별 매출액, 프로젝트별 진행률

#### 꺾은선 그래프 (Line Chart)
**사용 시기**:
- 시간에 따른 추세
- 연속적인 변화
- 여러 계열 비교

**예시**: 월별 실적 추이, 연도별 성장률

#### 원 그래프 (Pie Chart)
**사용 시기**:
- 전체 대비 비율
- 구성 요소 표현
- 4-6개 항목 이내

**예시**: 예산 배분, 시장 점유율

**주의**: 7개 이상 항목은 막대 그래프 권장

#### 누적 막대 그래프 (Stacked Bar)
**사용 시기**:
- 전체와 부분 동시 표현
- 카테고리별 구성 비교

**예시**: 분기별 매출 구성, 프로젝트 단계별 비용

#### 산점도 (Scatter Plot)
**사용 시기**:
- 두 변수 간 상관관계
- 분포 패턴 표현

**예시**: 예산 vs 기간, 품질 vs 비용

#### 간트 차트 (Gantt Chart)
**사용 시기**:
- 프로젝트 일정 관리
- 작업 진행 상황
- 마일스톤 표시

**예시**: 프로젝트 타임라인, 공정 계획

### 2. 차트 디자인 원칙
- **제목 명확히**: 차트가 무엇을 보여주는지 한눈에
- **범례 간결히**: 꼭 필요한 정보만
- **축 레이블**: X축, Y축 명칭과 단위 명시
- **색상 구분**: 계열별 명확한 색상 차이
- **격자선 최소화**: 너무 많으면 복잡함
- **핵심 강조**: 가장 중요한 데이터 색상 강조

### 3. 표 디자인 원칙
- **행/열 최소화**: 5행 × 5열 이내 권장
- **헤더 강조**: 배경색 또는 굵게
- **줄무늬**: 행 구분을 위한 교차 색상
- **정렬**: 숫자는 우측, 텍스트는 좌측
- **합계**: 굵게 또는 구분선

## 이미지 및 그래픽 가이드

### 1. 이미지 품질
- **해상도**: 최소 150 dpi, 권장 300 dpi
- **파일 형식**: PNG (투명 배경), JPG (사진)
- **크기**: 슬라이드의 40-70% 차지
- **압축**: 파일 크기 최적화 (10MB 이하)

### 2. 이미지 배치
- **여백 확보**: 이미지 주변 충분한 여백
- **정렬**: 슬라이드 중앙 또는 그리드 정렬
- **캡션**: 이미지 설명 또는 출처
- **테두리**: 필요시 얇은 테두리

### 3. 건축 도면 삽입
**준비 사항**:
- 도면 번호 확인
- 범례 포함
- 축척 표시
- 북쪽 방향 표시

**삽입 팁**:
- 전체 도면보다는 핵심 부분 확대
- 주석으로 중요 부분 표시
- 색상으로 강조 영역 표시
- 도면 제목과 날짜 명시

### 4. BIM 모델 이미지
**렌더링 설정**:
- 적절한 조명 설정
- 배경은 단색 또는 그라데이션
- 카메라 앵글 신중히 선택
- 그림자 활성화 (입체감)

**LOD 표현**:
- LOD 100: 개념적 매스 모델
- LOD 200: 일반적인 형상
- LOD 300: 상세한 디자인
- LOD 350: 시공 조정
- LOD 400: 제작 및 시공

### 5. 아이콘 사용
- **일관된 스타일**: 선형, 면형 중 하나로 통일
- **크기 통일**: 동일한 크기로 배치
- **색상**: 주 색상 또는 단색
- **간격**: 균등한 간격 유지

## 건축/BIM 프레젠테이션 특화 가이드

### 1. BIM 관련 발표
**필수 포함 요소**:
- BIM 소프트웨어명 및 버전
- IFC 버전 (데이터 교환 시)
- LOD 단계
- 협업 프로세스 (CDE 등)
- BIM 실행 계획 (BEP) 요약

**시각 자료**:
- 3D 모델 이미지
- 간섭 체크 결과
- 물량 산출 표
- 4D 시뮬레이션 캡처

### 2. 프로젝트 발표
**필수 정보**:
- 프로젝트명, 위치
- 발주처, 설계사, 시공사
- 규모 (연면적, 층수 등)
- 주요 일정 (착공, 준공)
- 예산 규모

**시각 자료**:
- 조감도, 투시도
- 배치도, 평면도
- 시공 단계별 사진
- 완공 사진 (해당 시)

### 3. 기술 발표
**구성**:
- 기술 개요 및 배경
- 핵심 기술 설명 (다이어그램)
- 기존 방식과 비교
- 적용 사례
- 기대 효과 (정량적)

**시각 자료**:
- 기술 흐름도
- 비교 표
- 적용 전후 사진
- 성과 지표 차트

### 4. 법규 및 기준 언급
**표기 방법**:
```
건축법 제48조 (용도변경)
KDS 41 00 00 (건축 설계 기준)
건축물의 에너지절약설계기준 제7조
```

**주의사항**:
- 법령명 정확히 기재
- 조항 번호 확인
- 최신 개정일 확인
- 관련 기준서 버전 명시

## 발표 팁

### 발표 전
1. **리허설**: 최소 2-3회 연습
2. **타이밍**: 발표 시간의 80%만 사용 (여유)
3. **장비 체크**: 프로젝터, 포인터, 마이크 등
4. **백업**: USB, 클라우드, 이메일 등 다중 백업

### 발표 중
1. **아이 컨택**: 청중과 시선 교류
2. **속도 조절**: 천천히, 또박또박
3. **포인터 사용**: 핵심 내용 강조
4. **질문 환영**: 적절한 시점에 질문 유도

### 발표 후
1. **Q&A**: 질문에 명확히 답변
2. **피드백**: 청중 반응 확인

## 자주 발생하는 실수

### 내용 관련
❌ **너무 많은 텍스트**: 슬라이드가 글로 가득 참
✅ **키워드 중심**: 핵심 단어만 표시

❌ **복잡한 차트**: 이해하기 어려운 복잡한 그래프
✅ **간단한 차트**: 한눈에 이해되는 시각화

❌ **저품질 이미지**: 흐릿하거나 픽셀화된 이미지
✅ **고품질 이미지**: 선명하고 명확한 이미지

### 디자인 관련
❌ **너무 많은 색상**: 무지개 같은 슬라이드
✅ **일관된 색상**: 3-4가지 색상 팔레트

❌ **작은 폰트**: 뒷자리에서 읽기 어려움
✅ **큰 폰트**: 최소 18pt 이상

❌ **과한 애니메이션**: 정신없는 효과
✅ **최소 애니메이션**: 필요한 곳에만

### 발표 관련
❌ **슬라이드 읽기**: 슬라이드 내용을 그대로 읽음
✅ **내용 설명**: 슬라이드는 보조, 설명이 주

❌ **청중 외면**: 화면만 보며 발표
✅ **청중 응시**: 청중과 눈 맞추며 발표

❌ **시간 초과**: 준비한 내용을 다 못 함
✅ **시간 준수**: 핵심 위주로 시간 내 마무리
